# Dry-Run Checklist

Before any real pilot use, the full starter pack must support a **fake dry-run**. No real client data is used at any point.

## The dry-run must prove

- [ ] Pilot folder structure works
- [ ] SQLite initializes and backs up
- [ ] Governance registry loads
- [ ] Kill switches default safe
- [ ] Evals pass
- [ ] Langfuse fake trace works
- [ ] Metadata-only mode is enforced
- [ ] Weekly proof template can be generated
- [ ] Readiness checklist can be completed
- [ ] Blake approval block exists
- [ ] No real client data is used

## Output

Save dry-run evidence (command output, screenshots, notes) in `12_audit_reports/` of the fake pilot folder and reference it in the deployment log.
